# PayFast — отчёт по безопасности и дизайн сети (Packet Tracer)

Этот репозиторий описывает **безопасный дизайн сети** для **PayFast** (финтех платформа платежей). Проект включает сегментацию по VLAN, защищённые сервисы (HTTPS + email), DMZ для публичного API сервера и site-to-site IPsec VPN между HQ и новым филиалом.

---

## Контекст компании
**Компания:** PayFast  
**Бизнес-модель:** комиссия за транзакции + premium подписка (финтех для малого e-commerce).  
**Критические сервисы:** payment gateway, merchant dashboard/reporting, fraud detection/monitoring, public API/SDK, settlement/reconciliation, KYC/onboarding.  

---

## Основные угрозы и фокус рисков
В дизайне учитываются high-risk угрозы:
- Утечка платежных данных
- Мошеннические транзакции / захват аккаунтов
- DDoS на API / потеря доступности
- Ransomware / компрометация инфраструктуры  
(а также insider threat, компрометация third-party и ошибки конфигурации).

**Оценка годового security-бюджета:** **$55,000**.

## Реализованные сервисы и меры защиты
### 1) Веб-сервер только по HTTPS (внутренний)
- IP веб-сервера: **192.168.20.10**
- HTTPS разрешён; HTTP ограничен

### 2) Защищённая email-инфраструктура (внутренняя)
Меры “best practice”:
- сложные пароли (админ + пользователи)
- ограничение портов (SMTP 25, POP3 110, IMAP 143)
- ACL на роутерах: разрешаем только нужный mail-трафик
- аутентификация отправки/получения писем

### 3) Новый филиал + site-to-site IPsec VPN
- Новый VLAN для серверов филиала: шлюз **192.168.70.1**
- IPsec VPN (AES для шифрования + SHA/HMAC для целостности)
- Pre-shared key для аутентификации роутеров
- Статические маршруты + crypto map для шифрования нужного трафика

### 4) DMZ + Zone-Based Policy Firewall (ZBF)
Публичный **Public API HTTPS server** размещён в **DMZ** и защищён ZBF:
- **INSIDE → OUTSIDE:** разрешаем доступ внутренних хостов к внешнему серверу (например ICMP для проверки)
- **OUTSIDE → INSIDE:** блокируем весь входящий трафик во внутренние VLAN

**Пример интерфейсов/зон**
- HQ Router: G0/0 (OUTSIDE/DMZ) **10.2.2.1**, G0/1 (INSIDE), S0/0/0 (VPN) **10.1.1.1**
- Branch Router: G0/0 (INSIDE) **192.168.50.1**, S0/0/0 (VPN) **10.1.1.2**
- DMZ HTTPS server: **10.2.2.2**

### 5) Wi‑Fi сеть (безопасно)
- Wireless VLAN **100** добавлен в INSIDE
- **WPA2-Enterprise** + **RADIUS сервер**
- Логин/пароль для каждого сотрудника (вместо общего Wi‑Fi пароля)

## Как проверить работу (рекомендуется)
### Связность и маршрутизация
- Ping внутри VLAN
- Ping между VLAN (по разрешённым сценариям)
- Проверка связи HQ ↔ Branch через VPN

### Команды на роутерах
```
show ip interface brief
show ip route
show crypto isakmp sa
show crypto ipsec sa
show running-config
```

### Проверка firewall (ZBF)
- OUTSIDE→INSIDE заблокирован
- INSIDE→OUTSIDE работает по разрешённым правилам
- Логи/статус (если есть в Packet Tracer)

### Проверка Web и Email
- HTTPS доступен, HTTP ограничен
- Email-порты доступны только из разрешённых VLAN
